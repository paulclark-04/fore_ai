## Airbus
- **CTO trap**: CTO at Airbus = physical product technology (wings, airframes), not software
- **"Engineering" means hardware**: Unless explicit software/IT context, engineering = airframe/mechanical
- **"Systems Engineering" = aircraft systems**: Avionics, hydraulics, flight controls — not software systems
- **"Test" = flight testing**: Physical component testing, not software testing. Verify context.
- **Good targets exist**: IT division (Head of IM, Cloud & DevOps), Innovation (Smart Factory), Digital Engineering
- **Key lesson**: Airbus has a massive IT division alongside its aerospace engineering. The IT people are great targets; the aerospace engineers are not.

## Boulanger
- **"Digital" usually means CX**: "Commerce Digital", "Expérience Digitale", "Exploitation Digitale" = customer experience/business roles
- **"Innovation" usually means product assortment**: "Innovation de l'Offre" = product category innovation, not tech
- **Business Unit directors**: Generic BU directors without tech scope = C tier
- **Good targets**: IT/tech team leads, Product Owners for web app (boulanger.com), Data officers

## Veepee
- **Strong QA team exists**: Lead QA Domain, SDET roles are textbook A-tier
- **Content Quality ≠ Software QA**: "Quality" in content/catalog context is C tier max
- **Revenue POs are D-tier**: Product Owners in revenue management/billing are commercial roles
- **Check current employment**: Some profiles in search results are former employees
- **Empty profiles exist**: Some employees have minimal LinkedIn presence — D tier, don't guess

## Groupe BPCE / BNP Paribas / Credit Agricole (Banking — General)
- **Executive sponsors are B-tier**: DG Technologies, DSI — pitch transformation not features
- **"Digital Fraud" ≠ Digital Transformation**: Cybersecurity/fraud divisions are separate budget
- **QA titles in banking = slam dunk**: Head of Software QA at a bank = highest scoring possible

## CA Personal Finance & Mobility (Sofinco)
- **Subsidiary of Crédit Agricole group**, consumer finance arm (Sofinco brand). Enterprise target.
- **Heavy "Product Manager/Lead" population**: Many are COMMERCIAL (financial product) roles, NOT software product roles. Critical to distinguish:
  - "Product Lead" on sofinco.fr web apps, with dev squads or "recette" mentions = technical → B+/A-tier
  - "Chef de produit" for crédit/assurance/offres commerciales = financial product manager → D-tier
  - "Product Manager Digital" optimizing "tunnels de conversion" = growth/UX role → B-tier max
- **Release Managers = textbook A-tier** (banking QA, direct Primary Buyer)
- **CIO/DSI at Sofinco = A-tier** (top IT decision maker, score 82-88)
- **"Services Numériques France Sofinco"** = internal IT department name, NOT an ESN. Don't flag as consultant.
- **Data roles are heavy**: CDO/CDAIO gets A-tier per CDO pattern. Data Governance Manager = C or D-tier. Data Scientist/Analyst = D-tier unless explicit software delivery scope.
- **AI Program Manager = A-tier champion** (Frederic VOIRIN pattern — mandate to integrate AI tools company-wide)
- **IT Cost Cutting + Purchasing = A-tier** (Jean-Hugues Seret pattern — cost reduction mandate + purchasing power, directly aligned with Fore AI's value prop)
- **Risk & Compliance IT**: Despite being "IT," these roles manage regulatory/audit systems, not customer-facing web apps. C-tier max, apply compliance penalty.
- **Same-title consistency**: Data Governance Managers should be scored consistently (both C or both D, not one of each)
- **Headline mismatch common**: Several employees have outdated headlines (e.g., "marketing digital" when current role is PO with dev squad). Always verify against enrichment data.
